## 网站文章排序修复

### 问题描述
- 博客页 (/blog/) 出现 500 错误: FieldError at /blog/ - Cannot resolve keyword 'date'
- 首页文章显示顺序错误，显示的是最早的6篇文章而非最新

### 根本原因
1. BlogIndexPage 的 get_context 方法使用 `.order_by('-date')` 在 Django ORM 层面排序，但 date 字段是 BlogPage 特有的，Page 模型没有
2. 首页使用的是 Django 视图 (views.py 中的 index 函数)，而非 Wagtail 页面系统。该视图直接查询文章但没有排序

### 解决方案
1. 将 ORM 排序改为 Python 内存排序：
   ```python
   posts = list(blog_index.get_children().specific().live())
   posts.sort(key=sort_key, reverse=True)  # sort_key 返回 post.date 或 post.first_published_at
   ```
2. 修改 views.py 的 index 函数，添加相同的排序逻辑

### 技术细节
- BlogPage 模型有 date 字段（发布日期），但 Wagtail 的 Page 模型没有
- 使用 hasattr 检查 date 属性是否存在，空值时用 first_published_at 作为后备
- 数据库中实际文章日期范围：2017-2020年，first_published_at 都是 2026-03-16（最近导入/发布）
